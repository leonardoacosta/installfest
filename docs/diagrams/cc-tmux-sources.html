Motion Animated tab icon

Real animation needs a wall-clock refresh, not a hook — hooks fire irregularly (a slow tool call can freeze the icon for 30s). window-status-format already re-evaluates on every status-interval tick (dropped to tmux's 1s floor) independent of hook activity, so the icon moved there instead of staying baked into the renamed text. Zero new background processes — same daemon-free design as the rest of this plugin.

Status Known gaps

IN PROGRESS
Usage percentages still stay "--" — root cause is now known and it's NOT nexus-agent failing to poll (that part is fixed, see below): Anthropic's /api/oauth/usage endpoint itself returns 429 rate_limit_error, even for a genuinely fresh token, confirmed via a direct live probe. Why: nexus-statusline (Claude Code's own registered statusLine command) independently calls this exact same endpoint on its own 5-minute file cache, completely uncoordinated with nexus-agent's poller (also 5 min) — the two compete for what's evidently a tight per-account rate limit. Decision made: consolidate to nexus-agent as the sole caller, with nexus-statusline reading the cached result instead of calling Anthropic itself. A design pass is in progress covering that consolidation plus a much larger ask — folding everything nexus-statusline currently shows (cost, lines added/removed, session clock, model, context %, token speed, git branch/worktree, spec, plus 5h/7d) into this SAME single-line tmux status-right segment, comprehensively enough that Claude Code's own statusline becomes unnecessary. Explicitly one bar, not a second tmux status row — that was considered and ruled out.
FIXED
Active credential's access token was permanently stale — active-credential-watcher.ts's dedupe gate only re-synced a credential into nexus-agent's pool on a fingerprint miss, but the fingerprint (SHA-256 of the refresh token) stays identical across an access-token-only refresh, so a credential's stale access token was written once on import and never updated again. Fixed in the nexus repo (dedupe gate made unconditional, plus a new proactive refresh job for every non-active pooled credential) and verified live: the active credential's expiresAt moved from months-stale to genuinely current immediately after deploy.
FIXED
Account label wasn't indicative — a bare name like "Leo" doesn't disambiguate when the same email can be authenticated against multiple orgs. Now full email + last char of the org id, e.g. leo@leonardoacosta.dev·e (usage.py:_account_label).
FIXED
Plugin marketplace pointed at a pre-rename path (~/dev/if, before the if→installfest move) — listed but failed to load: cache-miss. Re-registered at the correct path; install script hardened to check enabled status, not just presence, so this class of drift self-heals on future chezmoi apply.

Design Planned: usage consolidation + full-parity harvest

PLANNED — NOT YET IMPLEMENTED Designed (background plan pass, 28 tool-calls across both repos), not written. Two independent pipelines: one kills the 429 rate-limit competition, the other harvests everything Claude Code's own statusline shows today so cc-tmux can eventually supersede it. Explicitly ONE tmux bar — the status-right line the tabs already sit on — not a second status-format[1] row (that was the original plan; ruled out when asked "can we not have this be a separate bar?").
