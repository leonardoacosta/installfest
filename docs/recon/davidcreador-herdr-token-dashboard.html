STEAL effort: small herdr integration patterns: transcript reading, manifest shape, poll-primary gotcha
main.go · claude_test.go · herdr-plugin.toml · README.md
Before

No reference exists for locating/parsing a Claude Code session transcript, or which herdr manifest event hooks are trustworthy. cc-tmux has zero herdr awareness today.

After

docs/reference/herdr-integration-patterns.md captures the munge-cwd + glob fallback path resolution, the messageID+requestID turn dedup, the manifest shape, and — most load-bearing — that pane.agent_status_changed does not reliably fire on herdr 0.7.x, so any plugin must poll-and-diff as primary.

ADAPT effort: small Cost-estimation pricing table — longest-substring match + cache multipliers
main.go (claudePricing, claudeRates, claudeCost) · claude_test.go
Before

cc's claude-api skill has raw pricing figures but no documented recipe for estimating cost when a transcript has token counts but no embedded dollar figure — exactly Claude Code's own transcript shape.

After

Skill gains: match model substrings longest-first (so sonnet-4-5 beats sonnet-4 regardless of table order); bill cache-read at 0.1× and cache-write at 1.25× the matched input rate; unmatched models keep tokens but contribute $0 cost.

Novel Patterns

Genuinely novel, not adopted this run (no caller) — kept on record for the revisit trigger noted.

Agent-pane detection
Poll-loop-as-primary, event-hook-as-future-compat-stub — pane.agent_status_changed doesn't fire on herdr 0.7.0 per the plugin's own code comment; the README frames the manifest hook as kept for future herdr versions. Revisit: the moment any herdr plugin build starts here (installfest runs 0.7.5, same minor line).
Multi-agent data source
OpenCode live-API-with-disk-fallback — try 127.0.0.1:4096/session/{id} first, degrade to on-disk JSON when unreachable. Revisit: if OpenCode or a similar live+persisted agent CLI ever joins the fleet.
Docs tooling
ANSI-capture-to-SVG TUI preview — herdr pane read --source visible --ansi piped through an SGR-escape-code parser to generate an accurate README screenshot. Revisit: any future herdr/tmux plugin README needing a real preview image.
